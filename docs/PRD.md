# 產品需求文件 (PRD)

## 1. 專案概述
- **背景與動機**：對於喜歡下廚的家庭主婦和專業廚師來說，整理和尋找散落各處的食譜是一件繁瑣的事情。這個系統期望解決這個痛點，提供一個方便的集中化平台來收藏和管理他們喜愛的食譜。
- **目標用戶**：家庭主婦、廚師。
- **核心價值主張**：提供一個清晰、易用的線上食譜管理工具，讓使用者能輕鬆新增、搜尋並即時檢視詳細的材料與步驟，提升日常料理的效率與樂趣。

## 2. 功能需求
以下為系統的 5 項主要功能與對應的使用者故事：

- **功能一：新增食譜**
  - **使用者故事**：作為家庭主婦或廚師，我希望能夠新增食譜（包含名稱、材料清單、料理步驟），以便我將新學到或研究出的菜色記錄下來。
  
- **功能二：尋找食譜**
  - **使用者故事**：作為家庭主婦或廚師，我希望能夠透過關鍵字搜尋食譜，以便我在苦惱今天要煮什麼時，快速找到相關菜色。

- **功能三：檢視食譜明細（列出材料與步驟）**
  - **使用者故事**：作為家庭主婦或廚師，我希望能夠點擊單一食譜並查看詳細的材料與步驟，以便我在廚房料理時能直接跟著做。

- **功能四：編輯與刪除食譜**
  - **使用者故事**：作為家庭主婦或廚師，我希望能夠修改食譜內容或刪除不再需要的食譜，以便我隨時調整份量或修正錯誤，維持食譜清單的實用性。

- **功能五：建立食譜分類與標籤**
  - **使用者故事**：作為家庭主婦或廚師，我希望能夠替食譜加上分類或標籤（例如：中式、甜點、氣炸鍋），以便我能更有系統地整理及過濾收藏。

## 3. 非功能需求
- **技術限制**：專案將使用 Flask 作為開發框架，前端頁面使用 Jinja2 模板渲染，資料庫使用結構單純的 SQLite。
- **介面需求**：系統畫面需支援響應式設計（RWD），因為目標用戶經常會在廚房裡使用手機或平板來查看食譜。
- **效能與安全考量**：
  - 確保列表與搜尋食譜的載入時間順暢。
  - 表單輸入需進行基本驗證以防範 XSS 或 SQL Injection 攻擊。

## 4. MVP 範圍
- **Must Have (必須有)**
  - 新增食譜（輸入：名稱、材料、步驟）。
  - 食譜總覽與列表。
  - 列出單一食譜的詳細材料與步驟明細。
- **Should Have (應該有)**
  - 關鍵字搜尋食譜功能。
  - 編輯與刪除現有食譜。
  - 簡單的標籤分類機制。
- **Nice to Have (想要有)**
  - 食譜圖片上傳。
  - 最愛與釘選食譜功能。
  - 材料份量自動換算。
